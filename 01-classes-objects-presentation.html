    index = book_number - 1

              f"Publication Year: {book_publication_years[index]}")

        print("Book number does not exist.")
Better, but still fragile. What if lists get out of sync? No methods attached to data.

Class vs. Object

Understanding the difference

Wait... what's the difference?

Don't worry — this is one of the most common points of confusion when learning OOP. Let's break it down with analogies.

House Blueprint Analogy

  • The blueprint (class) isn't a house. It describes a house. It's the plan.
  • The house (object) is the real thing. It's built using the blueprint. It exists.
  • You can build many houses (objects) from the same blueprint.

Each house is different — different paint, different furniture (object attributes), but they all follow the same basic plan (class).

Class vs. Object

Class

Defines a type of thing

(e.g., a Student, a Car, a Dog)

"Student" is a type (class).

"Car" is a type (class).

Object

Is a specific instance of that type

(e.g., Alice, my red car, Buddy)

Alice is a specific student (object).

Volvo and BMW are specific cars (objects).

The Pizza Analogy

The Pizza class could define what all pizzas have in common (crust, sauce, cheese).

A specific pizza (pepperoni pizza, veggie pizza, hawaiian) is an object.

Class vs. Object

Feature Class (Blueprint/Template) Object (Instance/Real Thing)
Definition Blueprint or template for creating objects. Defines the type of thing. A specific instance of a class. A concrete thing created from the blueprint.
Creation Describes structure and behavior. Doesn't exist in memory until an object is created. Actual, concrete instance of the class. Occupies memory. Created using the class as a blueprint.
Purpose Defines the common properties and methods (behaviors) for all objects of that type. Represents a specific entity with its own unique data (values of the properties).
Attributes/Data Defines what kind of data objects of this class will have. Stores the actual values for the attributes defined by the class.
Methods/Behavior Defines what actions objects of this class can perform. Can perform the methods defined by the class.
Analogy A form or application template. A filled-out form or application.
Memory Doesn't occupy memory itself (just a description). Occupies memory to store its attributes and methods.
Number One class definition. Can have many objects (instances) of the same class.

Constructors prepare the new object for use, often accepting parameters that assign initial values to the object's attributes.

Understanding the __init__ Method

Book class with describe_book

class Book:
    def __init__(self, title, author, publication_year, genre, isbn, pages):
        self.title = title
        self.author = author
        self.publication_year = publication_year
        self.genre = genre
        self.isbn = isbn
        self.pages = pages

    def describe_book(self):
        description = f"{self.title} by {self.author}, " \
                      f"published in {self.publication_year}, " \
                      f"is a {self.genre} novel with ISBN {self.isbn} " \
                      f"and {self.pages} pages."
        print(description)

Constructors in Action: A Library Example

class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

class Library:
    def __init__(self):
        self.books = []

    def add_book(self, book):
        self.books.append(book)

my_library = Library()

my_book = Book("All the light we cannot see", "Anthony Doerr")
my_library.add_book(my_book)

my_library.add_book(Book("To Kill a Mockingbird", "Harper Lee"))
Objects can contain other objects! A Library holds a list of Book objects.

Understanding the self Parameter

def __init__(self, par1, par2, ...)

In Python, when we talk about self, we're talking about the individual instance of a class.

You can think of self as a way for an object to refer to itself.

self is the first parameter in any method defined within a class.

It must be included in the method definition so the method can access the instance's attributes and other methods.

Understanding the self Parameter

The mirror analogy

Imagine each object created from a class is a person.

Using self is like a person talking about themselves.

Just as a person knows their own thoughts and feelings, self knows the object's data and can access its methods and attributes.

Understanding the self Parameter

Person class example

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce_yourself(self):
        print(f"Hi, my name is {self.name} and I am {self.age} years old.")

# Creating two Person objects with the same attribute names but different values
person1 = Person("Alice", 30)
person2 = Person("Bob", 25)

person1.introduce_yourself()
# Outputs: Hi, my name is Alice and I am 30 years old.

person2.introduce_yourself()
# Outputs: Hi, my name is Bob and I am 25 years old.
When you call person1.introduce_yourself(), Python passes person1 as self automatically. That's how it knows which name and age to use.

Multiple Objects, Same Class

Each object is independent with its own attribute values

class Book:
    def __init__(self, title, author, year):
        self.title = title
        self.author = author
        self.year = year

    def describe(self):
        return f"{self.title} by {self.author} ({self.year})"

# Three different objects from the same class
book1 = Book("1984", "George Orwell", 1949)
book2 = Book("Dune", "Frank Herbert", 1965)
book3 = Book("The Hobbit", "J.R.R. Tolkien", 1937)

print(book1.describe())  # 1984 by George Orwell (1949)
print(book2.describe())  # Dune by Frank Herbert (1965)
print(book3.describe())  # The Hobbit by J.R.R. Tolkien (1937)
Each object stores its own data independently. Changing book1 does not affect book2 or book3.

How self Works Under the Hood

What Python does when you call a method

What You Write

person1 = Person("Alice", 30)
person1.introduce_yourself()

You call the method on the object using dot notation. No need to pass self yourself.

What Python Does

Person.introduce_yourself(person1)

Python automatically passes the object (person1) as the first argument (self) to the method.

This is why self is always the first parameter in every method definition. Python fills it in for you automatically.

class Book:

Why OOP?

Organization

Group related data and behavior together. Classes provide clear structure to your code.

Reusability

Define a class once, create as many objects as you need. No more copy-pasting code.

Real-World Modeling

Objects map naturally to real-world entities. Code becomes easier to understand and discuss.

Scalability

Adding new features means adding new classes or methods, not rewriting everything.
